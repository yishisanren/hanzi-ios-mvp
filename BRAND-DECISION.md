## 2. 决策推理（控制论建模）

### 系统建模：这是什么 App？

| 用户问题 | App 解决 |
|---------|---------|
| 看到字不知道怎么读 | → 拼音（带声调） |
| 听到字不知道怎么写 | → 笔顺动画 + 手写跟练 |
| 拼音 / 笔顺写错了不知道 | → 错误反馈 |
| 同音字打不出来 | → 语音说字选字 |

**核心动作 = 「查」+「写」**。光查不写 = Pleco；光写不查 = 字帖 App。本 App 的差异化在于"查写一体"。

### 名字必须命中的标准（前馈控制）

1. **同时覆盖「查」+「写」两个核心动作**，不可只占一边
2. **不需要被解释**（"声笔是什么？"用户问出这句话，名字就死了）
3. **大白话**（目标用户：成人学习者 + 家长，3-80 岁都能秒懂）
4. **可搜索性**（虽然主名不直接命中"拼音""笔顺"热词，但副标能补救）

### 历次废名复盘（反馈控制）

| 废名 | 主要问题 | 教训 |
|------|---------|------|
| 声笔 ToneStroke | 拼接合成词，用户看不懂 | 不要赌用户能解读"声=拼音、笔=笔顺" |
| 永字 / 起笔 / 一笔 / 临帖 / 字源 / 写字 / 一点 / 永 | 主控凭感觉拍，无推理链 | 必须从功能本身推 |
| 一格 / 八法 / 点墨 | 从 logo 锚点反推，偏美学 | 美学是 logo 的事，名字应该管功能 |
| 拼音笔顺 / 笔画笔顺 | 是功能术语不是品牌 | ASO 强但品牌无沉淀 |

**关键教训**：之前几轮失败都在"先有 logo 美学，再倒推名字"。**名字管"用户记什么"（功能），logo 管"用户认什么"（气质），两者不必字面对应**。

## 3. 为什么是「会读会写」

### 命中标准全过

| 标准 | 命中度 |
|------|:-----:|
| 覆盖「查」（=会读） | ✅ |
| 覆盖「写」（=会写） | ✅ |
| 不需要解释 | ✅ |
| 大白话 | ✅ |
| 可搜索性（配副标） | ✅ |

### 用户痛点即品牌名

成人汉字学习者最常说的两句话：
- "这字怎么念来着？" → 想要「会读」
- "我以为我会写，结果写不出来" → 想要「会写」

**「会读会写」就是用户痛点的直接镜像**。

### vs 头部竞品的差异化表达

| 竞品 | 它叫什么 | 我们的区别 |
|------|---------|----------|
| Pleco 汉语词典 | 字典型 | 我们不是字典，是"读+写"练习 |
| 写汉字 | 单核（写） | 我们双核（读+写） |
| 笔画笔顺 | 功能名 | 我们是品牌+功能双层 |
| 都会识字 / 洪恩识字 | 儿童识字 | 我们覆盖成人 + 双核 |
| Skritter | 写字学中文 | 我们更白话、不订阅 |

- 双层结构让品牌力 + 搜索力同时满足（这是 ASO 报告原本错过的解法）

## 5. 与已锁定 logo 的协同关系

**重要原则**：名字不必字面对应 logo。

- **Logo 管视觉记忆**：永字 + 米字格 + 某一笔朱砂红高亮
- **名字管功能记忆**：会读会写
- **两者关系**：logo 是品牌的视觉气质（东方极简 + 书写专业），名字是品牌的功能承诺（用户能得到什么）

如果硬要联结：**logo 里"永"字本身就是个汉字 → 用户"看到"（会读？）+ 笔顺动画 → 用户"写出来"（会写）**。视觉故事和功能承诺天然契合，但不必字面绑定。
