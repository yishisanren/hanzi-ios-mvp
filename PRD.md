# 汉字拼音与笔顺 iOS App MVP PRD

日期：2026-06-06（v2，结构性优化）
来源仓库：`yishisanren/hanzi-tools`
目标平台：iOS 17+，首版优先 iPhone
版本说明：v2 相对 v1 主要修复信息架构 / 离线策略 / 拼音方案三处自相矛盾，补齐多音字、License、埋点、iPad/Dark Mode/可访问性等缺失项；语音模块保持 v1 设计，待语音识别选型确认后再单独迭代。

## 1. 背景

`hanzi-tools` 当前是一个轻量 HTML/Node 工具，核心价值已经成立：输入汉字后展示拼音、笔顺动画，并提供练习书写模式。现阶段最适合的 iOS 迁移方式不是全量 Swift 重写，而是用 SwiftUI 承接 App 外壳和学习流程，用 `WKWebView` 复用成熟的 `hanzi-writer` 笔顺画布能力。

首版目标是把“能用的网页工具”升级成“可离线、可收藏、可持续学习的 iOS App MVP”。

## 3. 产品定位

产品名暂定：汉字练习 / Hanzi Tools / 声笔

一句话定位：一个面向汉字学习者的 iOS 拼音与笔顺练习工具，支持打字和语音说字，输入汉字即可查看读音、观看笔顺、跟写练习并保存学习记录。

目标用户：
- 成人汉字学习者：需要快速查拼音、看笔顺、练单字。
- 家长/低龄学习陪伴：需要简单、稳定、无广告的练字工具。
- 汉语教师/自学者：需要课堂或自学时快速演示笔顺。

非目标：
- 不做完整汉语课程。
- 不做 OCR 扫描识字。
- 不做复杂账号、社区、排行榜。
- 不做全量字典解释、组词、例句。

## 4. MVP 成功标准

必须满足：
- 用户能在 3 秒内完成一次汉字查询并看到拼音和笔顺画布（主路径为手动输入，语音作为差异化入口）。
- 用户能通过语音说出"张三的三"这类短语，并由 App 识别出目标汉字（语音作为亮点能力，不作为必经路径）。
- 用户能播放笔顺动画、进入练习书写、重置画布。
- 最近查询和收藏在本机持久保存。
- 首屏和练习页符合 iOS 手势与安全区布局。
- **完全离线可用**：首版 App Bundle 内置笔顺字形数据（覆盖 GB2312 一级字库 3755 字 + 二级字库 3008 字，共 6763 字），无需任何联网即可完成查询、播放笔顺、练习书写全流程。语音识别若依赖在线服务则单独标注，不影响其余核心路径离线运行。

建议指标：
- 首次启动到可输入时间 < 1.5 秒。
- 单字查询后画布可交互时间 < 2 秒。
- 最近查询记录保存最近 50 条。
- 收藏支持至少 200 个汉字或词组。

### 5.1 首页查询

用户故事：
用户打开 App 后，可以直接输入一个汉字或一个短词，点击查询后进入学习状态。

需求：
- 输入框支持中文输入，建议限制 1-4 个汉字。
- 空输入时给出轻量提示。
- 查询后展示：
  - 原始汉字/词组
  - 小写带调拼音
  - 每个汉字一个笔顺练习格
- 首页保留最近查询入口。

验收：
- 输入“爱”显示 `ài`。
- 输入“学习”显示 `xué xí`，显示两个练习格。
- 非汉字输入不崩溃，提示“请输入汉字”。

### 5.2 语音录入与说字选字

用户故事：
用户不想打字，或遇到同音字时，可以直接说“张三的三”“日出的出”“你是我的的我”“杜十娘的娘”“飞流直下三千尺的尺”，App 自动取出最后指定的目标字并进入学习页。

需求：
- 首页输入框右侧提供麦克风按钮。
- 首次使用时请求语音识别和麦克风权限。
- 语音识别结果先转为文本，再进入“说字解析器”。
- 解析器优先支持模式：`上下文短语 + 的 + 目标字`。
- 当一句话中有多个“的”时，默认取最后一个“的”后面的第一个汉字作为目标字。
- 兼容“的/得/地”被语音识别混淆的情况。
- 如果最后一个“的/得/地”后面不是汉字，则从整句中取最后一个汉字作为候选，并提示用户确认。
- 识别成功后，将目标字填入查询框并自动展示拼音与笔顺。
- 识别结果需要短暂显示原始听写文本，便于用户判断是否识别错。
- 语音失败、权限拒绝、没有识别出汉字时，回到手动输入，不阻塞主流程。

目标解析示例：
- “张三的三” -> “三”
- “日出的出” -> “出”
- “你是我的的我” -> “我”
- “杜十娘的娘” -> “娘”
- “飞流直下三千尺的尺” -> “尺”

验收：
- 点击麦克风后可录入上述 5 个示例，并进入对应单字学习页。
- “你是我的的我”必须解析为“我”，不能解析为“的”。
- 权限拒绝时显示可恢复提示，不导致页面空白或崩溃。
- 语音识别文字与最终选字都进入最近查询记录的元数据，便于后续排查。

### 5.3 笔顺动画

用户故事：
用户能观看汉字按正确顺序书写，理解每一笔的方向和顺序。

需求：
- 每个汉字使用 `hanzi-writer` 创建独立画布。
- 支持播放动画。
- 支持慢速演示。
- 支持重置。
- 多字词组按字顺序播放。

验收：
- “爱”创建 1 个 SVG 画布。
- “学习”创建 2 个 SVG 画布。
- 播放/慢速/重置按钮不会导致画布丢失或页面卡死。

### 5.4 练习书写

用户故事：
用户可以按笔顺在屏幕上跟写，App 对正确/错误笔画做反馈。

需求：
- 使用 `HanziWriter.quiz()` 进入练习模式。
- 错误笔画给出视觉反馈，不用弹窗打断。
- 完成一个字后自动进入下一个字。
- 完成词组后显示简短结果：完成、错误次数。

验收：
- 单字可进入 quiz。
- 多字按顺序 quiz。
- 练习完成后有完成反馈。

### 5.5 最近查询

用户故事：
用户可以快速回到最近练过的字。

需求：
- 本地保存最近 50 条查询。
- 去重：重复查询移到最前。
- 首页展示最近 8 条，其余进入列表。
- 点击记录直接打开学习页。

验收：
- 查询“爱”“学习”后，首页最近记录出现这两项。
- 重启 App 后记录仍在。

### 5.6 收藏

用户故事：
用户可以收藏想反复练习的汉字或词组。

需求：
- 学习页提供收藏/取消收藏。
- 收藏列表按最近收藏排序。
- 收藏项展示汉字和拼音。

验收：
- 收藏“爱”后，收藏页可见。
- 取消收藏后从收藏页消失。
- 重启 App 后状态保持。

### 5.7 设置

需求：
- 动画速度：正常 / 慢速。
- 书写格大小：跟随屏幕，默认自动。
- 外观：跟随系统 / 浅色 / 深色（见 §13）。
- 关于页：
  - 应用版本、构建号
  - 笔顺数据来源声明：hanzi-writer-data + Make Me a Hanzi + Arphic PL License **完整文本**

## 6. 技术方案

推荐架构：SwiftUI + WKWebView + 本地 Web 资源 + Swift/JS Bridge

SwiftUI 负责：
- App 导航结构。
- 首页输入。
- 语音录入入口、权限状态和识别结果确认。
- 最近查询和收藏。
- 设置。
- 本地数据持久化。

WKWebView 负责：
- `hanzi-writer` 画布。
- 笔顺动画。
- quiz 练习判定。

本地资源（完全离线策略）：
- 将 `hanzi-writer.min.js` 固定版本（v3.5）放入 App Bundle。
- 将当前 HTML 拆成可嵌入的 `writer.html`、`writer.js`、`writer.css`，所有外部依赖移除。
- **打包完整字形数据**：`hanzi-writer-data` 全集（GB2312 一二级共 6763 字 + 常用扩展，总计约 8-12MB 压缩后），通过自定义 `charDataLoader` 从 Bundle 内 JSON 加载，禁止运行时 fetch 任何远程资源。
- 字形数据来源：`hanzi-writer-data` (https://github.com/chanind/hanzi-writer-data)，基于 Make Me a Hanzi (Arphic PL License)，详见 §10 License 合规处理。

拼音方案（单一来源原则）：
- **唯一拼音源：Swift 层**，使用 `pinyin4swift` 或基于 `pypinyin` 词库移植的本地实现。
- WebView 层不再输出拼音，只负责画笔顺；旧 `index.js` 中的 `cnchar` 拼音逻辑在 iOS 版本中完全废弃。
- 输出规范：小写带调（ài / xué xí），音节间空格分隔，多字默认按词组连写匹配规则。
- 多音字处理见 §6.5。

Bridge 设计：
- Swift -> JS：
  - `loadCharacters(chars)`（拼音由 Swift 单独显示，不再传给 JS）
  - `animate(speed)`
  - `startQuiz()`
  - `reset()`
- JS -> Swift：
  - `writerReady`
  - `quizMistake`
  - `quizCorrectStroke`
  - `quizComplete`
  - `loadError`

本地数据：
- `UserDefaults` 可满足 MVP 的最近查询和设置。
- 收藏建议用 `SwiftData` 或轻量 JSON 文件；若工程目标 iOS 17+，优先 SwiftData。

### 6.5 多音字处理

策略：
- **单字查询**：默认显示最高频读音；若该字存在多音，在拼音右侧加小字"另读 xxx"灰色提示，点击可切换。
- **词组查询**：按词组词典匹配读音（如"中国" zhōngguó、"中奖" zhòngjiǎng、"重点" zhòngdiǎn、"重复" chóngfù）。词典未命中时退化为逐字最高频。
- **数据源**：Swift 层拼音库必须支持词组模式（pypinyin 已有，pinyin4swift 需确认）；如果选型的 Swift 库不支持词组，则 M0 阶段需要补一份高频词组拼音表（建议从 pypinyin `phrases-dict` 移植 Top 5000 词组）。
- **验收**：单字"行" → 默认 xíng + "另读 háng"；词组"银行" → yínháng（不出现 xíng）；词组"中国" → zhōngguó。

语音方案：
- 使用 iOS 系统语音识别能力作为首版入口（`SFSpeechRecognizer` iOS 17+ 离线模式）。
- 若 TestFlight 阶段发现说字短语识别率低于 80%，按 ASR-RESEARCH.md 调研报告切换到 `sherpa-onnx` (Zipformer-zh) 端上方案。
- 语音结果不直接当作最终查询词，必须经过说字解析器。
- 说字解析器应作为独立 Swift 纯函数实现，便于单元测试（详细 spec 见 voice-parser-spec.md，本版暂不展开）。
- 首版先支持普通话识别；其他方言和英文解释不进入 MVP。

## 7. 信息架构

单主页面结构：
- 顶部：品牌、设置入口。
- **第一优先级：手动输入框**（中文键盘默认聚焦），右侧带麦克风按钮启动语音录入。
- **第二优先级：语音说字**作为输入框的次级入口，承担"不方便打字 / 不会打字 / 同音字困扰"场景。
- 识别结果区（语音触发后才出现）：原始听写文本、最终选字、拼音、开始练习、播放。
- 快捷区：最近练习、收藏入口、说法示例（仅语音首次使用时引导一次）。

学习状态结构：
- 选字后仍留在主页面展开学习状态。
- 中部显示汉字/词组、拼音、笔顺练习格。
- 操作按钮包括开始练习、播放、慢速、重置、收藏。
- 首版不使用底部 Tab，收藏 / 最近 / 设置 / 关于 均作为主页面顶部或次级页面进入。

## 8. 品牌与 UI 方向

Logo 方向图：`assets/logo-directions.png`

### 推荐方向：声笔 ToneStroke

推荐理由：
- 同时覆盖“拼音声调”和“落笔练习”两个核心功能。
- iOS 小图标识别度高。
- 比纯教材风更有品牌记忆点，比科技风更贴合汉字学习。

主题色：
- 主墨黑：`#141414`
- 朱砂红：`#D93A2E`
- 宣纸白：`#F6F1E6`
- 暖灰：`#A8A099`

UI 风格：
- 东方极简。
- 宣纸白背景，大面积留白。
- 控件克制，强调练习区。
- 朱砂红用于主要行动、声调、当前笔画反馈。
- 墨黑用于正文、汉字、线条。

首版组件风格：
- 练习格：宣纸白底 + 暖灰细线。
- 主按钮：朱砂红填充，白字。
- 次按钮：浅灰或描边。
- 卡片半径：8-12pt，避免过度圆润。
- 字体：系统字体为主，汉字展示可用系统宋/楷风格备选，但不依赖第三方字体。

备选方向：
- `GlyphPulse`：更现代，适合后续强调动画轨迹和练习反馈。
- `OneGrid`：最稳定教材感，适合低风险上架。
- `InkTone`：更温润亲子，但图形意象略装饰。

风险：`hanzi-writer-data` 字形数据基于 Arphic PL License（Make Me a Hanzi 派生），上架 App Store 必须显式声明。
处理：
- 在 App 内"关于"页**必须**展示完整 Arphic PL License 文本 + Make Me a Hanzi 原作者署名（Shaunak Kishore）+ hanzi-writer-data 项目链接。
- App Store 描述末尾加致谢段落。
- M0 阶段先让工程师完整阅读 Arphic PL License，确认是否需要随 App 一并提供源数据下载链接（Arphic PL 要求保留许可证副本但不要求开源整个 App）。
- 同步检查 `pinyin4swift` / 移植的 `pypinyin` 词库 License（pypinyin 用 MIT，pinyin4swift 待查）。

风险：字形数据完全打包带来 App Bundle 体积上升（预计 +10MB）。
处理：MVP 选用 GB2312 一二级 6763 字（覆盖 99% 日常使用），生僻字（Unicode 扩展 A/B 区）作为 P2 按需下载。Bundle 总体目标控制在 < 25MB。

风险：拼音库 JS/Swift 表现不一致。
处理：MVP 已收口为 **Swift 层单一拼音源**，WebView 不再输出拼音。多音字策略见 §6.5。

风险：WebView 练习手势与 SwiftUI 滚动冲突。
处理：练习页画布区域禁止父级滚动抢手势，底部按钮固定。

风险：多字词组画布在小屏拥挤。
处理：1-2 字并排，3-4 字横向分页或纵向列表。

风险：语音识别可能把"的"识别成"得/地"，或把诗句识别成近音字。
处理：首版加入候选确认；解析器兼容"的/得/地"，并允许用户一键改字。

风险：MVP 缺少观测能力，TestFlight 阶段难以复盘问题。
处理：见 §12 最小本地埋点方案。

风险：iPad 用户下载后体验不佳影响 App Store 评分。
处理：见 §13 设备适配。

## 12. 最小本地埋点

目标：MVP 阶段在不收集任何用户数据的前提下，让 TestFlight 反馈可复盘。

设计：
- **完全本地**：所有日志写入 App Documents/logs/，从不上传，从不联网。
- **滚动日志**：保留最近 7 天，每天一个文件，超期自动删除，总大小不超过 5MB。
- **用户主动导出**：设置 → 反馈 → "导出诊断日志"按钮，触发系统分享面板，用户自己决定是否发邮件给开发者。

记录字段（JSON Lines 格式）：
- 时间戳（本地时间，精度到秒）
- 事件类型：`query` / `voice_input` / `voice_parse` / `practice_start` / `practice_complete` / `error`
- 查询字 / 词组（仅 query 事件）
- 语音原文 → 解析结果 → 用户是否手动改字（仅 voice 事件）
- 错误码与堆栈摘要（仅 error 事件，不含 PII）
- 网络状态（在线 / 离线）

隐私边界：
- 不记录设备 ID、IDFA、IP、位置。
- 不记录收藏列表（用户隐私）。
- 不记录任何账号信息（MVP 本就无账号）。
- 隐私清单中声明 "App Functionality" 用途。

## 13. 设备与外观适配

### iPad 适配（最小支持）

策略：MVP 不做 iPad 专属布局，但确保**可用**：
- Info.plist 声明支持 iPhone + iPad（不设 `UIRequiresFullScreen`）。
- 布局采用 SwiftUI 响应式（GeometryReader / 自适应宽度），iPad 上以单列居中显示，左右最大留白。
- 练习格在 iPad 上按 iPhone Pro Max 尺寸渲染并居中，不强行拉伸。
- 验收：iPad mini 6 / iPad Air M2 竖屏、横屏均能完成核心查询练习流程。

P2 计划：iPad 专属侧栏布局（左侧最近+收藏列表，右侧学习区）。

### Dark Mode

策略：跟随系统 + 用户可强制。
- 主墨黑 `#141414` → Dark Mode 反转为 `#F6F1E6`（宣纸白，作为前景文字）。
- 宣纸白 `#F6F1E6` → Dark Mode 反转为 `#1A1A1A`（深灰，作为背景）。
- 朱砂红 `#D93A2E` 不变，作为强调色在两种模式下都可见。
- 暖灰 `#A8A099` → Dark Mode 调整为 `#5C5550`（练习格细线）。
- 验收：iOS 设置切换浅色 / 深色，App 实时跟随，无白屏闪烁。

### 可访问性（最小集）

MVP 范围：
- 所有按钮支持 VoiceOver label（如"麦克风，开始语音录入"）。
- 拼音文字用 VoiceOver 朗读时按汉字朗读（不读拉丁字母），交由系统中文 TTS。
- 笔顺画布对 VoiceOver 用户暂不优化，关于页声明"笔顺动画为视觉功能"。
- 支持 Dynamic Type（输入框、最近查询、菜单字号跟随系统）。
- 主按钮对比度满足 WCAG AA（朱砂红 + 白字已通过）。

不进入 MVP：
- 笔顺画布 VoiceOver 描述（"第 X 笔，横"）—— P2。
- 全键盘导航 —— P2。

### 错误状态兜底

| 场景 | 表现 |
|------|------|
| WebView 加载失败 | 显示"练习页加载失败，点击重试"占位卡片 + 重试按钮 |
| Bridge 调用超时（>3s） | Toast 提示"加载较慢，已重试" + 后台自动 reload |
| 磁盘空间不足（写收藏） | Alert "存储空间不足，请清理后再试"，不损坏现有数据 |
| SwiftData 迁移失败 | 启动检测，若失败用旧版数据 fallback + 上报到本地日志 |
| 输入非汉字字符 | 输入框下方红色细字提示"请输入汉字"，不弹窗 |
| 字形数据缺失（生僻字） | 显示拼音 + "笔顺数据暂未收录"，不崩溃 |
